# sturdy-pancake
MÃ DỰ ÁN: UDM_17

1. Đề xuất Dự án: Ứng dụng Quản lý và Truyền tải dữ liệu qua Mạng nội bộ (Desktop Application)

Bạn muốn làm gì: Xây dựng một ứng dụng Desktop (giao diện trực quan) hỗ trợ các máy tính trong cùng mạng nội bộ kết nối, nhắn tin và truyền tải các tệp tin dữ liệu qua lại với nhau một cách an toàn và nhanh chóng.

Tính năng đang cố gắng bổ sung/hoàn thiện: Giao diện người dùng trực quan, dễ kết nối qua IP và Cổng (Port).
  - Tính năng mã hóa dữ liệu cơ bản khi truyền tải để đảm bảo an toàn.
  - Quản lý lịch sử các tệp tin đã gửi và nhận thành công.

Công nghệ sử dụng: Ngôn ngữ lập trình phù hợp cho ứng dụng Desktop (như C# WinForms/WPF hoặc Java Swing/FX), kiến trúc Client-Server, sử dụng giao thức kết nối Socket (TCP/IP).

Mục tiêu cuối cùng: Tạo ra một sản phẩm hoàn chỉnh, không sử dụng dòng lệnh, không chạy trên trình duyệt web, đáp ứng tốt nhu cầu trao đổi dữ liệu nội bộ và tối ưu hóa hiệu suất lập trình mạng.

2. Kế hoạch thực hiện (Mỗi 2 tuần)

Giai đoạn 1 (Tuần 1 - Tuần 2): Khởi tạo dự án, thiết kế sơ đồ kiến trúc hệ thống và xây dựng giao diện UI cơ bản cho ứng dụng Desktop.

Giai đoạn 2 (Tuần 3 - Tuần 4): Lập trình các tính năng kết nối mạng cốt lõi, thiết lập kết nối Socket Client-Server để truyền văn bản (Chat).

Giai đoạn 3 (Tuần 5 - Tuần 6): Hoàn thiện tính năng truyền/nhận file dung lượng lớn và xử lý tối ưu hóa băng thông mạng.

Giai đoạn 4 (Tuần 7 - Tuần 8): Kiểm thử hệ thống, sửa lỗi kết nối, tối ưu hóa giao diện và đóng gói ứng dụng sản phẩm hoàn chỉnh.
